# Study Vault — Computer Science & Software Engineering

A personal knowledge base of CS and software-engineering notes, organized as a linear curriculum. The eleven top-level domains are **numbered to suggest a reading order**, roughly building from first principles up to applied topics:

> foundations → systems → data → distribution → design → performance → quality → practice → applied tech

## How notes are organized

- **Each folder has a `README.md` index** linking to its notes in a recommended reading order.
- Notes come in two states:
  - **Written notes** — explained in prose, in a concise academic style.
  - **Study outlines** — a structured list of headings for a topic not yet written up. Fill these in as you study; the heading order already follows a fundamentals → mechanics → trade-offs → advanced progression.
- Some sections include a **`review-questions.md`** note for self-testing.

## Conventions

- One topic per file; filenames are kebab-case, titles are Title Case.
- This is both an Obsidian vault and a git repo, so links use standard Markdown (`[text](path.md)`) which works in both.
